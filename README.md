# Telegram JTBD-консультант

Telegram-бот на базе LLM, выступающий в роли эксперта по методологии **Jobs-to-be-Done (JTBD)**. Помогает предпринимателям и продакт-менеджерам находить идеи для продуктов, формулировать job stories, выявлять необслуженные работы и валидировать продуктовые гипотезы — через привычный интерфейс Telegram, без установки дополнительных приложений.

## Возможности

- **JTBD-консультация** — бот задаёт уточняющие вопросы и помогает структурировать мышление в рамках фреймворка JTBD
- **Ответы через LLM** — каждое сообщение обрабатывается языковой моделью через OpenRouter API
- **Настраиваемая роль** — системный промпт задаётся через переменную окружения, без правок в коде
- **Команды `/start` и `/help`** — краткое приветствие и описание возможностей
- **Только личный чат** — бот отвечает исключительно в приватных диалогах
- **Деплой в одну кнопку** — Railway автоматически собирает образ и запускает сервис после `git push`

## Установка и запуск

### Требования

- macOS
- [uv](https://docs.astral.sh/uv/) — менеджер зависимостей Python
- Docker (опционально, для запуска в контейнере)

### 1. Клонирование и настройка окружения

```bash
git clone <repo-url>
cd <repo-dir>
cp .env.example .env
```

Заполните `.env` своими значениями:

| Переменная | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота из [@BotFather](https://t.me/BotFather) |
| `OPENROUTER_API_KEY` | Ключ API с [openrouter.ai](https://openrouter.ai) |
| `OPENROUTER_MODEL` | Модель, например `openai/gpt-4o-mini` |
| `SYSTEM_PROMPT` | Системный промпт (роль JTBD-консультанта) |

Опциональные переменные:

| Переменная | По умолчанию | Описание |
|---|---|---|
| `OPENROUTER_BASE_URL` | `https://openrouter.ai/api/v1` | Базовый URL API |
| `LLM_TEMPERATURE` | значение в коде | Температура генерации |
| `LLM_MAX_TOKENS` | значение в коде | Лимит токенов ответа |
| `LOG_LEVEL` | `INFO` | Уровень логирования |

### 2. Запуск без Docker

```bash
make install   # установить зависимости (uv sync)
make run       # запустить бота
```

### 3. Запуск в Docker

```bash
make docker-build   # собрать образ
make docker-run     # запустить контейнер
```

## Деплой в Railway

1. Зарегистрируйтесь на [railway.app](https://railway.app) и создайте новый проект.
2. Подключите репозиторий: **New Project → Deploy from GitHub repo**.
3. В Railway Dashboard → **Variables** добавьте обязательные переменные окружения (те же, что и в `.env`).
4. Railway автоматически обнаружит `railway.toml` и `Dockerfile`, соберёт образ и запустит бота.
5. При каждом `git push` Railway пересобирает образ и перезапускает сервис.
6. Логи доступны в Railway Dashboard → вкладка **Logs**.

> Открытый порт не нужен — бот работает на long polling без HTTP-сервера.

## Заметки

- История диалога хранится в памяти процесса и сбрасывается при перезапуске бота.
- Не запускайте несколько экземпляров бота одновременно — Telegram не поддерживает параллельный long polling.
- Сообщения не из личного чата (`private`) бот игнорирует.
