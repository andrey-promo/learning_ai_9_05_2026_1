# ADR 0002: Python как язык реализации

## Статус

Принято

## Дата

2026-05-09

## Контекст

Нужен язык для серверной логики бота, работы с сетью (Telegram, API LLM) и быстрой итерации. Команда ориентируется на экосистему, где уже зафиксирован менеджер окружений **uv** (см. [ADR 0001](./0001-use-uv-for-python.md)).

## Решение

Основной язык реализации проекта — **Python**.

Целевая ветка интерпретатора на старте: **Python 3.11+** (совместимость с типизацией и современным стандартной библиотекой; точная минимальная версия может быть уточнена при появлении `pyproject.toml`).

## Последствия

### Положительные

- Богатая экосистема для ботов и HTTP-клиентов.
- Совпадает с выбором **uv** для зависимостей и окружений.

### Отрицательные / риски

- Необходимо явно закреплять версию Python в документации и CI (когда они появятся).
