# ADR 0001: Использование uv для Python-проекта

## Статус

Принято

## Дата

2026-05-09

## Контекст

Проект предполагает разработку на Python (Telegram-бот с LLM). Для воспроизводимых сборок нужен инструмент, который:

- ставит зависимости быстро и предсказуемо;
- управляет виртуальным окружением;
- упрощает онбординг (один понятный вход вместо связки pip + venv + отдельных утилит).

## Решение

Зафиксировано использование **uv** (Astral) как основного инструмента для установки пакетов и управления окружениями Python в этом репозитории.

## Последствия

### Положительные

- Быстрая установка и разрешение зависимостей.
- Единый инструмент вместо разрозненных команд и сценариев.
- Активная экосистема и совместимость с привычным миром PyPI.

### Отрицательные / риски

- Участникам команды нужно один раз установить `uv` и следовать принятому в проекте формату файлов (например, `pyproject.toml` / lock-файл — по мере появления в репозитории).
- При миграции с Poetry/pip-tools потребуется явная процедура переноса зависимостей.
