# ADR 0003: Telegram как канал взаимодействия с пользователем

## Статус

Принято

## Дата

2026-05-09

## Контекст

По [описанию идеи](../idea.md) ассистент должен быть доступен как **Telegram-бот**, текстовый диалог, без отдельного клиентского приложения.

## Решение

Пользовательский канал — **Telegram**. Интеграция строится на **Telegram Bot API** (официальный HTTP API Telegram).

Конкретная Python-библиотека-обёртка над Bot API **не зафиксирована в этом ADR** и выбирается при реализации (её можно оформить отдельным ADR, если появятся требования к async, масштабированию webhook vs long polling и т.п.).

## Последствия

### Положительные

- Знакомый пользователям канал, не нужна своя витрина приложений.
- Документированная модель безопасности через токен бота.

### Отрицательные / риски

- Ограничения платформы (лимиты сообщений, модерация, политика Telegram).
- Нужны надёжное хранение токена бота и учёт нагрузки (режим обновлений: webhook или long polling).
