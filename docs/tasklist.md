## Итерации

### Итерация 1 — Скелет, конфиг, живой бот
> Цель: бот запускается, читает конфиг и отвечает на команды.

- [ ] Создать `pyproject.toml` с зависимостями (aiogram, openai, python-dotenv)
- [ ] Создать структуру `src/telegram_llm_bot/` с пустыми модулями
- [ ] Создать `.env.example` с перечнем всех переменных
- [ ] Реализовать `config.py` — загрузка и валидация env, логирование
- [ ] Реализовать `main.py` — инициализация Bot, Dispatcher, запуск polling
- [ ] Реализовать `handlers.py` — `/start`, `/help`, заглушка на нетекст
- [ ] Убедиться: `/start` возвращает ответ в личном чате

---

### Итерация 2 — Подключение LLM (без истории)
> Цель: бот отвечает на текст через OpenRouter, каждый запрос независим.

- [ ] Реализовать `llm_client.py` — обёртка над OpenAI-клиентом с `base_url` OpenRouter
- [ ] Подключить `llm_client` в хендлер текстовых сообщений
- [ ] Убедиться: пользователь пишет текст → получает ответ от модели

---

### Итерация 3 — История сообщений в памяти
> Цель: бот помнит контекст диалога в рамках сессии (без БД).

- [ ] Хранить последние N сообщений по `user_id` в памяти процесса
- [ ] Передавать историю в запрос к LLM вместе с системным промптом
- [ ] Убедиться: модель учитывает предыдущие реплики в разговоре

---

### Итерация 4 — Обработка edge-cases
> Цель: бот ведёт себя корректно в нестандартных ситуациях.

- [ ] Ошибка LLM/сети → нейтральный ответ пользователю + лог с деталями
- [ ] Сообщения не из личного чата — игнорировать без лога
- [ ] Убедиться: каждый сценарий из раздела «Сценарии работы» vision.md работает

---

### Итерация 5 — Docker и Makefile
> Цель: бот запускается в одну команду как без Docker, так и в контейнере.

- [ ] Написать `Dockerfile` (Python 3.12, зависимости через uv)
- [ ] Написать `Makefile` с целями: `install`, `run`, `docker-build`, `docker-run`
- [ ] Написать `README.md` с инструкцией по запуску
- [ ] Убедиться: `make run` и `make docker-run` поднимают рабочего бота
